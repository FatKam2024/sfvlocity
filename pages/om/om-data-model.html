OM Data Model

Understand the core data structure of Salesforce Vlocity Order Management to effectively work with and customize the system.

Introduction

The OM data model is the foundation of how order information is stored and related within the Salesforce Vlocity platform. Understanding this model is crucial for efficient system use and customization.

Key Objects

  • Order: Represents a customer's request for products or services.
  • Order Item: Individual line items within an order.
  • Product: Catalog of available products and services.
  • Price List: Contains pricing information for products.
  • Account: Customer information linked to orders.

Hands-On Lab: Exploring the OM Data Model

This lab will guide you through the process of exploring and understanding the OM data model in a Salesforce org.

Lab Steps

  1. Log into your Salesforce developer org.
  2. Navigate to Setup > Object Manager.
  3. Explore the Order, Order Item, and Product objects.
  4. Create a simple ERD (Entity Relationship Diagram) of these objects.
  5. Create test records to understand how the objects relate to each other.

Best Practices for Working with the OM Data Model

  • Always consider the relationships between objects when querying or updating data.
  • Use the standard objects and fields whenever possible before creating custom ones.
  • Regularly review and optimize your data model to ensure it meets evolving business needs.
