Customization, Configuration & Basic Setup in Salesforce Vlocity

This comprehensive guide covers the essential setup, configuration, and customization techniques in Salesforce Vlocity CPQ, providing the foundational knowledge necessary for effective platform management.

Introduction

Customization and configuration are key to tailoring Salesforce Vlocity CPQ to your business needs. This document also includes basic setup tasks, which lay the groundwork for more advanced customization and configuration efforts.

Basic Configuration Tasks

This section covers the essential configuration tasks required to set up Salesforce Vlocity. These foundational tasks ensure that your environment is ready for customization and further development.

  • Setting Up User Roles and Permissions: Configuring roles to ensure that users have appropriate access.
  • Initial System Settings: Configuring basic system settings to align with your business needs.
  • Data Import/Export: Basic tasks to import and export data efficiently.
  • Basic Customization: Adding custom fields and modifying page layouts.

Customization & Configuration

After completing the basic setup, you can proceed to more advanced customization tasks, such as creating custom objects, implementing validation rules, and setting up workflow rules.

Key Areas of Customization

  • Custom Fields: Adding fields to capture additional data.
  • Page Layouts: Adjusting the layout of record pages to suit user preferences.
  • Validation Rules: Implementing rules to ensure data integrity.
  • Workflow Rules: Automating business processes with workflows.
  • Custom Objects: Creating new objects to represent unique business entities.

Hands-On Lab: Customization & Configuration in CPQ

This lab will guide you through essential customization and configuration tasks in Salesforce Vlocity CPQ, providing hands-on experience to reinforce your understanding.

Lab Steps

  1. Log into your Salesforce org with Vlocity CPQ installed.
  2. Create a custom field on a standard object to capture additional data.
  3. Modify a page layout to include the new custom field.
  4. Set up a validation rule to enforce data entry requirements.
  5. Create a workflow rule to automate a common business process.
  6. Implement a hands-on configuration task to practice your skills.

Best Practices for Customization & Configuration

  • Start with declarative (point-and-click) tools before resorting to custom code.
  • Document all customizations for easier maintenance and troubleshooting.
  • Test all customizations in a sandbox environment before deploying to production.
  • Regularly review and optimize customizations to align with changing business needs.
  • Train users on the new customizations to ensure they are used effectively.
